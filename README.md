# WordPress 异步图片批量上传工具

## 📖 项目概述

这是一个专为 **写真站** 设计的高性能 WordPress 图片批量上传工具。它能够自动扫描本地文件夹中的图片，批量上传到 WordPress 媒体库，并自动创建包含所有图片的草稿文章。

### 核心特性

| 特性 | 说明 |
|------|------|
| 🚀 **异步高性能** | 基于 `aiohttp` 实现异步并发上传，速度提升 3-5 倍 |
| 🔒 **数据完整性** | 多重校验机制，防止重复上传、图片缺失、文件名错乱 |
| 📊 **智能并发** | 根据上传成功率动态调整并发数，自适应网络状况 |
| 💾 **断点续传** | SQLite 数据库记录上传状态，支持中断后继续 |
| 🏷️ **SEO 优化** | 自动生成 alt 文本、标题等 SEO 友好的元数据 |
| 📁 **批量处理** | 一键处理多个文件夹，自动创建对应的草稿文章 |

### 适用场景

- 写真图集网站批量发布
- 摄影作品集上传
- 图片画廊网站内容管理
- 任何需要批量上传图片到 WordPress 的场景

## 📦 安装步骤

### 1. 环境要求

- **Python**: 3.8 或更高版本
- **操作系统**: Windows / macOS / Linux
- **WordPress**: 5.0+ (需启用 REST API)

### 2. 安装依赖

```bash
# 创建虚拟环境（推荐）
python -m venv venv

# 激活虚拟环境
# Windows:
venv\Scripts\activate
# macOS/Linux:
source venv/bin/activate

# 安装依赖包
pip install aiohttp pillow tqdm
```

### 3. WordPress 配置

#### 3.1 创建应用密码

1. 登录 WordPress 后台
2. 进入 **用户** → **个人资料**
3. 滚动到底部找到 **应用密码**
4. 输入名称（如 "图片上传工具"）
5. 点击 **添加新的应用密码**
6. **复制生成的密码**（只显示一次）

![应用密码示例](https://developer.wordpress.org/news/wp-content/uploads/2020/11/application-passwords-user-profile.png)

#### 3.2 确保 REST API 可用

访问 `https://你的域名/wp-json/wp/v2/users/me`，如果返回 JSON 数据则表示 API 正常。

### 4. 配置脚本

打开脚本文件，修改 `Config` 类中的配置：

```python
class Config:
    # WordPress配置 - 必须修改
    WP_DOMAIN = "your-domain.com"           # 你的WordPress域名
    WP_USER = "your-username"               # WordPress用户名
    WP_APP_PASSWORD = "xxxx xxxx xxxx xxxx" # 应用密码（保留空格）
    
    # 本地目录 - 必须修改
    BASE_DIRECTORY = r"C:\你的图片目录"      # 包含图片文件夹的目录
```

## ❓ 常见问题解答

### Q1: 连接测试失败

**症状**: `✗ 认证失败: 401`

**解决方案**:
1. 检查用户名是否正确
2. 检查应用密码是否正确（注意保留空格）
3. 确认 WordPress REST API 已启用
4. 检查是否有安全插件阻止 API 访问

```python
# 测试配置是否正确
WP_USER = "your-username"  # 不是邮箱，是用户名
WP_APP_PASSWORD = "xxxx xxxx xxxx xxxx xxxx xxxx"  # 保留空格
```

### Q2: 上传速度很慢

**可能原因**:
1. 服务器带宽限制
2. 并发数设置过低
3. 图片文件过大

**解决方案**:
```python
# 增加并发数
MAX_CONCURRENT_UPLOADS = 20
INITIAL_CONCURRENT = 12

# 减少延迟
REQUEST_DELAY = 0.02
BATCH_DELAY = 0.05
```

### Q3: 出现 "文件过大" 错误

**解决方案**:
1. 检查 WordPress 的上传限制（媒体 → 设置）
2. 修改 `php.ini`:
   ```ini
   upload_max_filesize = 64M
   post_max_size = 64M
   ```
3. 或在上传前压缩图片

### Q4: 数据库错误 "no such column"

**原因**: 旧版本数据库与新版本不兼容

**解决方案**: 程序会自动迁移，如果仍有问题：
```bash
# 备份后删除数据库文件
mv wp_upload_history.db wp_upload_history.db.bak
# 重新运行程序会自动创建新数据库
```

### Q5: 图片顺序错乱

**原因**: 文件名排序问题

**解决方案**: 
- 确保文件名使用数字前缀：`001.jpg`, `002.jpg`
- 或使用自然排序友好的命名

### Q6: 部分图片上传失败

**查看日志**:
```bash
# 查看错误日志
cat wp_uploader.log | grep ERROR
```

**重新运行**: 程序会自动跳过已上传的图片，只处理失败的

### Q7: 如何只处理特定文件夹？

修改代码临时指定：

```python
async def main():
    processor = FolderProcessor()
    await processor.uploader.test_connection()
    
    # 只处理特定文件夹
    await processor.process_folder(r"C:\Pictures\写真预览\特定文件夹")
```

### Q8: 如何清除上传记录重新开始？

```python
# 删除数据库中特定文件夹的记录
import sqlite3
conn = sqlite3.connect("wp_upload_history.db")
conn.execute("DELETE FROM uploaded_images WHERE folder_name = '文件夹名'")
conn.execute("DELETE FROM created_posts WHERE folder_name = '文件夹名'")
conn.commit()
```
